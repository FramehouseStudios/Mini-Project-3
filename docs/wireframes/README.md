# MarketFlow Wireframes

Submitted by Joshua Ojeda.

This folder separates the wireframes by route so the design process is easy to review during grading.

## Page Wireframes

- [Dashboard](dashboard.md) - landing page, project summary, category rail, featured products
- [Products](products.md) - search, filter, sort, API product grid
- [Product Detail](product-detail.md) - dynamic route, product information, interactive review form
- [Planner](planner.md) - saved-product shortlist using Context state
- [About](about.md) - architecture and presentation notes

## Review Flow

1. Start with the Dashboard wireframe to understand the overall app purpose.
2. Move to Products to see the main interaction model.
3. Open Product Detail to show dynamic routing and form handling.
4. Open Planner to show shared Context state.
5. Finish with About to review architecture and requirements coverage.
