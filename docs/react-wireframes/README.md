# Blockbuster+ Route Wireframes

Submitted by Joshua Ojeda

Each application route has a separate wireframe so the design process is easy to compare with the final React implementation.

## Page Wireframes

- [Dashboard](dashboard.md) - project summary, API statistics, and featured films
- [Films](films.md) - URL-backed search, filter, sort, and API film grid
- [Film Detail](film-detail.md) - dynamic route, film information, and controlled review form
- [Rental Bag](rental-bag.md) - persistent Context state shared between routes
- [Catalog Manager](catalog-manager.md) - controlled CRUD form and database records
- [About](about.md) - architecture and presentation notes

The matching visual SVGs use the same numbered route order. `7-key-and-hierarchy.svg` documents the component tree.

## Review Flow

1. Start with Dashboard to establish the product and live data source.
2. Move to Films to demonstrate the primary interaction model.
3. Open Film Detail to show dynamic routing and controlled review state.
4. Open Rental Bag to show shared Context and localStorage persistence.
5. Use Catalog Manager to demonstrate complete database CRUD.
6. Finish with About and Swagger to summarize the architecture.
